A vault is an encrypted file that contains all your secrets, projects, environments, and mappings. Everything is stored as a single encrypted blob — nothing is ever written to disk in plaintext.

## Vault Basics

Each vault is a standalone `.envly` file that can be stored anywhere on your computer. When you unlock a vault, its contents are decrypted into memory. When you lock it (or close the app), everything is securely wiped from memory.

Envly supports **multiple vaults** through a vault registry. You might have separate vaults for:
- Work projects
- Personal projects
- A specific client

You can switch between vaults at any time. Switching automatically locks the current vault and selects the new one.

## Creating a Vault

When creating a vault, you choose:

- **Name** — a label to identify the vault
- **File path** — where to save the encrypted file
- **Cipher mode** — how the vault is protected:
  - **Passphrase** — you enter a master password to unlock. The password is stretched using Argon2id (a memory-hard key derivation function) before encrypting with XChaCha20-Poly1305.
  - **Keychain** — a random encryption key is generated and stored in your OS keychain (macOS Keychain, Windows Credential Manager, Linux Secret Service). The vault unlocks automatically when your OS session is authenticated.

## Importing a Vault

If you already have a vault file (e.g. shared by a colleague or restored from a backup), you can import it by selecting the file. Envly reads the cipher mode from the file header and adds it to your registry.

## Exporting a Vault

You can export an encrypted copy of your vault to a different location with a separate passphrase. This is useful for:
- Creating backups
- Sharing a vault with someone else (they'll need the export passphrase to open it)

The export always uses passphrase encryption regardless of your vault's current cipher mode.

## Vault Lifecycle

A vault goes through three states:

| State | Description |
|-------|-------------|
| **Uninitialized** | No vault file exists yet. You're guided through the creation wizard. |
| **Locked** | The vault file exists but is encrypted. You need to authenticate to access it. |
| **Unlocked** | The vault is decrypted and in memory. You can manage secrets and projects. |

When the vault locks (via your action or when the app exits):
1. All active environments are deactivated (symlinks and temp files are cleaned up)
2. Changes are saved to disk
3. Decrypted data is wiped from memory

## Changing Your Passphrase

If your vault uses passphrase mode, you can change the master password from **Settings**. The vault is re-encrypted with the new passphrase.

## Switching Cipher Mode

You can migrate between passphrase and keychain modes from **Settings**. Envly re-encrypts the vault with the new cipher and updates the registry.

## Vault Safety

- **Atomic writes** — vault saves use a temp-file-then-rename strategy so a crash mid-write never corrupts your vault
- **Automatic backups** — before each save, the previous vault file is copied to a `.bak` file as a crash-safety mechanism
- **Stale pruning** — on startup, Envly removes registry entries for vault files that no longer exist on disk
