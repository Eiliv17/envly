Secrets are the core data in Envly. Each secret is a globally defined key-value pair stored inside your encrypted vault.

## Secret Properties

| Property | Required | Description |
|----------|----------|-------------|
| **Key** | Yes | A unique identifier, e.g. `POSTGRES_PASSWORD_STAGING` |
| **Value** | Yes | The sensitive data. Encrypted at rest, masked in the UI by default. |
| **Description** | No | A human-readable note about what the secret is for |
| **Tags** | No | Labels for filtering and organization (e.g. `database`, `staging`, `aws`) |
| **Expiration date** | No | When set, the UI shows warnings as the date approaches |

## Managing Secrets

### Creating Secrets

From the **Secrets** page, click the create button and fill in the key, value, and any optional fields. Secret keys must be unique within a vault.

### Editing Secrets

Click on a secret to edit any of its properties. If you change the value of a secret that's currently in use by an active environment, Envly automatically refreshes the symlink so your running dev server picks up the new value.

### Revealing Values

Secret values are masked by default in the UI. Click the reveal button to temporarily show a secret's value. Values are only sent from the backend to the frontend on demand — they're not included in list responses.

### Deleting Secrets

You can delete a secret only if it's not referenced by any environment mapping. If it is, Envly will tell you which environments depend on it so you can clean up the references first.

## Tags

Tags help you organize and filter secrets. Add any number of tags to a secret (e.g. `database`, `staging`, `aws`, `stripe`). On the Secrets page, you can filter the list by tag to quickly find related secrets.

## Expiration Dates

Set an optional expiration date on a secret to remind yourself to rotate it. Envly shows:
- A warning when a secret is approaching its expiration
- An alert when a secret has expired

This encourages regular secret rotation without you having to remember dates manually.

## Bulk Import

If you already have secrets in `.env` files, you can import them in bulk:

1. Open the bulk import dialog from the Secrets page
2. Select a `.env` file from your machine
3. Envly parses the file and shows you all the key-value pairs it found
4. Review and confirm — new secrets are created, and if keys already exist you can choose to update them

This is especially useful when you're first migrating to Envly from a manual `.env` file workflow.

## Referential Integrity

Envly enforces referential integrity between secrets and environment mappings:

- A secret cannot be deleted while any mapping references it
- When a secret's value changes, all active environments using it are automatically refreshed
- If a bulk import updates a secret that's currently active, the symlink is refreshed immediately
