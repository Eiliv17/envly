## Download

Head over to the [Downloads](/downloads) page to grab the latest release for your platform.

| Platform | Format | Architecture |
|----------|--------|-------------|
| macOS | `.dmg` | Apple Silicon (ARM64), Intel (x86_64) |
| Windows | `.exe` | x86_64 |
| Linux | `.AppImage` | x86_64 |

You can also find all releases on the [GitHub releases page](https://github.com/eiliv17/envly/releases).

## macOS

1. Download the `.dmg` file for your architecture (Apple Silicon or Intel).
2. Open the `.dmg` and drag **Envly** into your Applications folder.
3. Launch Envly from Applications. On first launch, macOS may ask you to confirm since the app is from an identified developer.

## Windows

1. Download the `.exe` installer.
2. Run the installer and follow the prompts.
3. Launch Envly from the Start menu.

### Symlink Permissions

Envly uses symlinks to inject environment variables. On Windows, creating symlinks requires either **Administrator privileges** or **Developer Mode** to be enabled. Envly checks for this at startup and will guide you if it's not available.

To enable Developer Mode:
1. Open **Settings** > **Update & Security** > **For developers**
2. Toggle **Developer Mode** on

## Linux

1. Download the `.AppImage` file.
2. Make it executable: `chmod +x envly-*.AppImage`
3. Run it: `./envly-*.AppImage`

::tip
On Linux, you can move the AppImage to `~/.local/bin/` or `/opt/` for easier access.
::

## Building from Source

If you prefer to build Envly yourself:

### Prerequisites

- [Rust](https://rustup.rs/) (stable toolchain)
- [Node.js](https://nodejs.org/) >= 18
- [pnpm](https://pnpm.io/) >= 10
- Platform-specific Tauri dependencies ([see Tauri docs](https://v2.tauri.app/start/prerequisites/))

### Steps

```bash
git clone https://github.com/eiliv17/envly.git
cd envly
pnpm install
pnpm dev:desktop
```

To create a production build:

```bash
pnpm build:desktop
```
