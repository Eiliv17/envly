A project in Envly maps to a single repository or working directory on your machine. Each project holds environments that define which secrets get injected and how.

## Project Properties

| Property | Required | Description |
|----------|----------|-------------|
| **Name** | Yes | A label for the project (e.g. "API Backend", "Mobile App") |
| **Folder path** | Yes | The local path to your project directory |
| **Env filename** | No | The file to create in the project directory. Defaults to `.env` but can be set to `.env.local`, `.env.development`, etc. |
| **Description** | No | A note about the project |
| **Starred** | No | Pin a project to the top of the dashboard for quick access |

## Creating a Project

From the **Projects** page, click create and fill in the name and folder path. Envly includes a native folder picker so you can browse to the directory. The env filename defaults to `.env` but you can change it to match your framework's convention.

## Dashboard

The **Dashboard** provides a quick overview of all your projects. Each project card shows its environments with activation toggles. Starred projects appear first for quick access.

From the dashboard you can:
- Activate or deactivate environments with a single click
- Star/unstar projects
- Navigate to a project's detail page

## Folder Path Validation

Every time the app starts or a project is loaded, Envly checks whether the folder path still exists on disk. If it doesn't (e.g. you moved or deleted the repository), the project is flagged with a warning so you know it needs attention.

## Cloning Projects

You can clone a project to create a copy with all its environments and mappings. This is useful when you have similar setups across repositories — clone the project, update the folder path, and you're ready to go.

## Env Filename

Different frameworks look for different files:
- **Most tools**: `.env`
- **Next.js / Nuxt**: `.env.local`
- **Rails**: `.env.development`
- **Docker Compose**: `.env`

Set the env filename per project to match what your tools expect. When an environment is activated, the symlink is created with this filename in the project directory.
